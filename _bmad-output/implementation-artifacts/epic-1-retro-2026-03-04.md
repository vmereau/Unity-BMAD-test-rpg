## Challenges & Growth Areas

### Recurring Code Review Patterns (caught in review every time)

1. **OnDisable null guard** (HIGH severity)
   - `_input` initialized in `OnEnable`, but `OnDisable` can fire before `OnEnable` if the component is disabled in `Awake`
   - Caught in Story 1.2 as HIGH severity; proactively added in Story 1.3; documented in CLAUDE.md
   - Required pattern: `if (_input == null) return;` at top of every `OnDisable` that uses `_input`

2. **CharacterController velocity magnitude includes Y** (MEDIUM severity)
   - `CharacterController.velocity.magnitude` always returns ~2.0f when grounded (GROUNDED_VELOCITY = -2f)
   - Initial Story 1.4 implementation used raw magnitude — blend tree never reached idle threshold
   - Caught in code review and fixed to horizontal-only: `new Vector3(v.x, 0, v.z).magnitude`
   - Documented in CLAUDE.md

3. **Incomplete `.meta` files** (MEDIUM severity)
   - New scripts created without proper `MonoImporter` block in `.meta`
   - Caught in Story 1.4 and Story 1.5 code reviews
   - Documented in CLAUDE.md

4. **Raw input device access** (MEDIUM severity, Story 1.2 only)
   - Initial `CameraController` used `Mouse.current` and `Keyboard.current` instead of Input System action maps
   - Fixed to `_input.UI.Cancel` / `_input.UI.Click`
   - Documented in CLAUDE.md

### One-Off Significant Blocker

- **InputSystem_Actions cross-assembly reference** (Story 1.5) — Adding `Game.asmdef` moved all `_Game/` scripts to the `Game` assembly, but `InputSystem_Actions.cs` was at `Assets/` root in `Assembly-CSharp`. Named assemblies cannot reference `Assembly-CSharp`. Fix: moved the generated file into `Assets/_Game/`. Now documented in CLAUDE.md.

## Key Insights & Learnings

1. **Read CLAUDE.md before writing any MonoBehaviour** — The checklist of recurring patterns (null guards, velocity, meta files) is already there. If dev agent reads it proactively, code review becomes a confirmation rather than a catch-all.

2. **Cinemachine 3.x differs significantly from 2.x** — `CinemachineCamera` (not `CinemachineVirtualCamera`), use `CinemachineSameAsFollowTarget` for OTS aim, never `CinemachineRotationComposer`. All documented in CLAUDE.md.

3. **Jump was missing from the movement vocabulary** — Retrospective caught that `Jump` is wired in the Input System but was never implemented. Added as Story 1.6 (critical before Epic 2).

## Significant Discoveries

### Jump Mechanic Missing (Affects Epic 2 Readiness)

**Discovery:** The `Jump` action exists in the Player input map but was never implemented in `PlayerController` or `PlayerAnimator`. No jump velocity, no jump animation.

**Impact:** Epic 2 combat feel (dodge roll, combat positioning) depends on complete movement vocabulary. Starting combat stories without jump would leave the movement foundation incomplete.

**Resolution:** Added as **Story 1.6** — to be implemented and completed before Epic 2 begins. Story should follow the exact patterns of Stories 1.3 and 1.4:
- `jumpHeight` field added to `PlayerConfigSO`
- Jump velocity applied in `PlayerController` on `Jump` action press when `isGrounded`
- Jump/fall/land animation clips from Mixamo, added to `PlayerAnimatorController` blend tree
- `PlayerAnimator` drives the jump state from `CharacterController.isGrounded` + vertical velocity
